# Contributing

## Workflow

1. Open or link an issue.
2. Create a focused branch.
3. Update skill content with clear scope boundaries.
4. Include validation notes in the PR.
5. Request review from CODEOWNERS.

## PR Requirements

- Linked issue
- Clear problem statement
- Before/after behavior
- Verification steps
- No overlap with existing skill ownership

## Quality Bar

- Bounded outputs
- Explicit stop conditions
- Deterministic output contracts
- Capability-first routing where needed
